Tecnologias e decisões:
Nome do projeto: chatbot-whatsapp-atestados

Chatbot WhatsApp para coleta de atestados médicos.
Substitui formulário manual por conversa guiada no WhatsApp.
Sem interface web — toda interação via WhatsApp Business API (Meta Cloud API).

Backend: Node.js 20+ / TypeScript / Express
Handler → Service → Prisma → Validator (Zod)
Logging com console estruturado (JSON em produção)

ORM: Prisma (schema.prisma + migrations)
Seed: números autorizados admin/user de teste

Banco: PostgreSQL 16

Autorização por número de telefone — tabela authorized_numbers
Roles: ADMIN (exporta Excel) / USER (envia atestados)
Sem JWT, sem login web — identidade pelo número do WhatsApp

Integração: Meta Cloud API (WhatsApp Business)
Webhook recebe mensagens → Express processa → responde via API
Segurança: verificação X-Hub-Signature-256 em todo POST

Docker Compose: PostgreSQL + API (Node.js)
Ambiente dev: ngrok para tunnel HTTPS ao webhook
Multi-stage build para produção

Validação: Zod para toda entrada do usuário
Excel: ExcelJS para gerar planilha de atestados (/exportar, role ADMIN)
Testes: Vitest

Git: repositório único
Branches: develop + main
Local por enquanto

Backend fica em /api
Prisma schema em /api/prisma
Documentos do projeto ficam em /documentos
Regras de desenvolvimento em /regras

Sem frontend web
Sem Nginx
Sem CDNs ou dependências externas em runtime
Commits não precisam de padrão
